fun convertPositiveOffsetToNegativeOffset(value: String): String
Content copied to clipboard

this looks to see if there is an "all zero offset" preceded by a plus sign on the date time value. if there is, then we're going to flip this bit over to be a negative offset. Note, according to the ISO-8601 specification, UTC is NEVER supposed to be represented by -0000, only ever +0000. That said, RFC3339 does offer the opportunity to use -0000 to reflect an "unknown offset time", so it is still valid and does parse. Also, Java understands and can parse from -0000 to +0000, so we are not breaking our implementation there.
